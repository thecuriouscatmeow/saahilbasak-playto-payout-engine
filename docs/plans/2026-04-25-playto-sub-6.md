## Goal
Plain Vite + React + TS + Tailwind dashboard. Layered (api / hooks / components / features / utils). Polling at 3s with tab-hidden pause. Idempotency-key auto-injected on POST. No component library.

## Acceptance Criteria
1. `npm run dev` boots the dashboard against the running backend.
2. Selecting a merchant displays balance (available + held) in INR (Indian numbering: ₹1,23,456.78).
3. Submitting a valid payout returns success and the new payout appears in the history table within 6 seconds (one poll cycle).
4. Insufficient-balance attempt shows a clear error.
5. Submitting twice rapidly with the same form data still shows two payouts in history (each gets a fresh `Idempotency-Key`).
6. Hiding the browser tab pauses polling; reactivating resumes immediately.
7. "Collected in USD · Paid out in INR" label visible on dashboard.
8. No `fetch` call inside any file under `src/components/`.
9. No JSX inside any file under `src/hooks/` or `src/api/`.

### M6.2 — API client layer
- `src/api/client.ts`:
  - `fetchJson<T>(path, init?)`: prepends `VITE_API_BASE_URL`, sets `Content-Type: application/json`, parses JSON, throws typed `ApiError(status, body)`.
  - On `init.method === 'POST'` and no `Idempotency-Key` set: inject `crypto.randomUUID()`. Allow caller to pass one (used by retries). Also auto-injects `X-Merchant-Id` from a getter passed at module init (set during merchant selection).
- `src/api/types.ts`: TS interfaces matching every response shape in spec §12.
- `src/api/payoutsApi.ts`: `createPayout`, `listPayouts(merchantId)`, `getPayout(id)`.
- `src/api/merchantsApi.ts`: `listMerchants`, `getBalance(id)`, `getTransactions(id)`, `getBankAccounts(id)`.
- **Commit:** `feat(frontend): api client + typed endpoints + idempotency-key injection`

### M6.3 — Utils
- `src/utils/formatInr.ts::formatInr(paise: number): string`:
  - Convert paise → rupees (integer division for major, modulo for minor).
  - Indian grouping: last 3 digits, then 2 at a time. Example: 1234567890 paise → `₹1,23,45,678.90`.
  - Implement via `Intl.NumberFormat('en-IN', { style: 'currency', currency: 'INR', maximumFractionDigits: 2 })` and verify output format. Fall back to manual grouping if Intl emits unexpected separators.
- `src/utils/formatTimestamp.ts::formatRelative(iso: string)`, `formatAbsolute(iso: string)`.
- `src/utils/uuid.ts::generateUuid()` (wraps `crypto.randomUUID`).
- `src/utils/constants.ts`: `POLL_INTERVAL_MS = 3000`, `MERCHANT_STORAGE_KEY = 'playto_merchant_id'`.
- **Test:** `frontend/src/utils/formatInr.test.ts` (Vitest) — Indian numbering for ₹1, ₹100, ₹1234, ₹123456, ₹12345678.90.
- **Commit:** `feat(frontend): utils — Indian INR formatting + timestamp + uuid`

### M6.4 — Hooks
- `src/hooks/usePolling.ts`: per spec §3.2 example. `useEffect`-based, pauses on `document.hidden`, fires immediately on resume.
- `src/hooks/useMerchant.ts`: reads/writes `localStorage[MERCHANT_STORAGE_KEY]`; exposes `merchantId, setMerchantId`. On mount, ensures the API client's `X-Merchant-Id` getter returns the current value.
- `src/hooks/useBalance.ts`: state + polling for `getBalance(merchantId)`.
- `src/hooks/usePayouts.ts`: state + polling for `listPayouts(merchantId)`.
- `src/hooks/useTransactions.ts`: one-shot fetch, expose `refetch`.
- `src/hooks/useCreatePayout.ts`: mutation hook returning `{ submit, loading, error, lastResult }`. On success, refetches balance + payouts hooks (via callback or shared store — simplest: pass refetch callbacks in).
- **Commit:** `feat(frontend): hooks — polling, balance, payouts, transactions, create-payout, merchant`

### M6.5 — Presentational components
- `src/components/MoneyText.tsx`: `<MoneyText paise={n} />` rendering `formatInr(n)`. Optional `tone="muted"` for held balance.
- `src/components/StatusBadge.tsx`: pill rendering for `pending` (amber), `processing` (blue), `completed` (green), `failed` (red). Tailwind utility classes.
- `src/components/BalanceCard.tsx`: card with `available` + `held` from props.
- `src/components/PayoutRow.tsx`: table row.
- `src/components/TransactionRow.tsx`: ledger row with type chip + amount + reference.
- `src/components/TableShell.tsx`: header + row container.
- `src/components/FormField.tsx`: label + input + error slot.
- `src/components/Button.tsx`: primary/secondary variants.
- All components are pure: props in, JSX out, callbacks for events. **Forbidden:** `useEffect` for data, `fetch`, `useState` for fetched data.
- **Commit:** `feat(frontend): presentational components — pure props/JSX, no I/O`

### M6.6 — Feature components + App wiring
- `src/features/MerchantSelector.tsx`: dropdown of merchants (uses `merchantsApi.listMerchants` via inline fetch on mount; this is fine for one-shot list), persists selection via `useMerchant`.
- `src/features/DashboardPanel.tsx`: composes `BalanceCard` + ledger summary; uses `useBalance`.
- `src/features/PayoutForm.tsx`: rupee input, bank account dropdown, submit button. Converts rupees → paise before posting. Uses `useCreatePayout`. Surfaces 422 / 409 errors clearly.
- `src/features/PayoutHistorySection.tsx`: `TableShell` + `PayoutRow` × N from `usePayouts`. Includes header with "Last updated Xs ago".
- `src/features/TransactionLedger.tsx`: `TableShell` + `TransactionRow` × N from `useTransactions`.
- `src/App.tsx`: minimal layout — header with "Playto · Collected in USD · Paid out in INR", `MerchantSelector`, then once a merchant is selected: `DashboardPanel`, `PayoutForm`, `PayoutHistorySection`, `TransactionLedger`. No router.
- **Commit:** `feat(frontend): feature components + dashboard wiring`
