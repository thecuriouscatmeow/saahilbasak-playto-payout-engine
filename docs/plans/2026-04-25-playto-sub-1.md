## Goal
Stand up the Django project with the full layered folder skeleton, all domain primitives, all data models with constraints, and a working seed command. Nothing in this phase opens an HTTP route — it lays the foundation every later phase drops code into.

## Acceptance Criteria
1. `python manage.py migrate && python manage.py seed` runs cleanly against real PostgreSQL.
2. Folder structure matches spec §13.1 exactly (empty `__init__.py` files placed where modules will live).
3. Unit tests in `tests/unit/` pass: `test_domain_transitions.py`, `test_money_helpers.py`, `test_request_hash.py`.
4. `pytest` runs against real PG via `DATABASE_URL` (no sqlite anywhere).
5. All DB `CHECK` constraints from spec §4 enforce on insert (verified by a test).

### M1.4 — Models + migrations + DB constraints
- `apps/merchants/models.py`: `Merchant`, `BankAccount` per spec §4.1.
- `apps/payouts/models.py`: `Transaction`, `Payout`, `PayoutEvent`, `IdempotencyRecord` per spec §4.1.
- All amounts `BigIntegerField`. UUIDs as PKs (`uuid.uuid4` default). `timestamptz` via `auto_now_add` / `auto_now` where appropriate.
- DB-level `CHECK` constraints via `Meta.constraints` with `CheckConstraint`:
  - `Transaction`: `amount_paise > 0`, `type IN (...)`, `(type='credit') = (payout_id IS NULL)`.
  - `Payout`: `amount_paise > 0`, `status IN (...)`.
  - `IdempotencyRecord`: `state IN (...)`, `UNIQUE(merchant_id, idempotency_key)` via `UniqueConstraint`.
- Indexes per spec §4.1: `(merchant_id, created_at DESC)` on transactions and payouts; `(payout_id)` on transactions; partial `(status, last_attempted_at) WHERE status='processing'` on payouts; `(payout_id, created_at)` on payout_events; `(expires_at)` on idempotency_records.
- Generate migrations: `python manage.py makemigrations`.
- **Test:** add `tests/integration/test_db_constraints.py` — assert `IntegrityError` on negative amount, invalid type, completed-credit-with-payout-id mismatch, duplicate (merchant_id, idempotency_key).
- **Commit:** `feat(models): merchants + payouts schema with DB-level constraints`
