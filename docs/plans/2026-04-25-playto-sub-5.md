## Goal
Wire structured logging with correlation IDs threaded API → worker → DB. Implement the reconciliation service + management command that asserts ledger ↔ payout invariants. Add the senior-signal stress harness.

## Acceptance Criteria
1. JSON logs to stdout with `correlation_id`, `payout_id`, `merchant_id`, `idempotency_key`, `attempt_number` populated where applicable.
2. `jq 'select(.payout_id == "<id>")' logs.jsonl` reconstructs a single payout's full lifecycle in chronological order.
3. `python manage.py reconcile` exits 0 on seeded + post-test data, exits 1 with a drift report when invariants violated.
4. `python manage.py stress_concurrency --merchants 3 --workers 20 --duration 10` runs to completion and asserts global ledger invariant.
5. `test_reconciliation` passes — drift detection on injected mismatched row.

## Milestones

### M5.1 — `structlog` configuration
- `observability/logging.py`:
  ```python
  import structlog
  structlog.configure(
      processors=[
          structlog.contextvars.merge_contextvars,
          structlog.processors.add_log_level,
          structlog.processors.TimeStamper(fmt="iso"),
          structlog.processors.JSONRenderer(),
      ],
      wrapper_class=structlog.make_filtering_bound_logger(logging.INFO),
  )
  ```
- Helpers: `bind_correlation_id(cid)` contextmanager wrapping `structlog.contextvars.bind_contextvars`/`unbind_contextvars`.
- `config/settings/base.py`: route stdlib logging to structlog; `LOGGING` config minimal.
- **Test:** `tests/unit/test_logging.py` — log a message inside `bind_correlation_id`, capture stdout, parse JSON, assert `correlation_id` present.
- **Commit:** `feat(observability): structlog JSON config + correlation_id contextvars`

### M5.2 — Request-scoped correlation middleware
- `observability/middleware.py::CorrelationIdMiddleware`:
  - On request: read `X-Correlation-Id` header or generate `uuid.uuid4()`. Bind via `bind_contextvars(correlation_id=...)`.
  - On response: set `X-Correlation-Id` header.
  - On finally: `clear_contextvars()`.
- Add to `MIDDLEWARE` list in `base.py` (early — before logging middleware if any).
- **Test:** `tests/integration/test_correlation_middleware.py` — POST a request, assert response header, assert log lines from the request carry the header value.
- **Commit:** `feat(observability): correlation_id middleware`

### M5.3 — Wire log events at call sites
Emit events from spec §15.2 with appropriate level + context:
- `CreatePayoutService`: `payout.created`, `idempotency.replayed`, `idempotency.in_flight`, `idempotency.payload_mismatch`.
- `ProcessPayoutService`: nothing direct — relies on `transition()` events.
- `payout_repo.transition()`: emit `payout.{to_status}` with `payout_id`, `from`, `to`, `attempt_number`.
- `txn_repo.insert_release()`: emit `payout.funds_released`.
- `RetryStalePayoutsService`: emit `payout.retry_scheduled` per payout, `payout.max_attempts_exceeded` on max-attempts path.
- `CreatePayoutService` middleware-side: `request.start`/`request.end` already from middleware (M5.2).
- All events use `structlog.get_logger().info(event_name, **fields)`.
- **Test:** `tests/integration/test_log_events.py` — capture stdout (`pytest --capture=tee-sys` or capsys), POST a payout, assert each expected event name present with required fields.
- **Commit:** `feat(observability): emit canonical log events at call sites`

### M5.4 — `ReconcileLedgerService` + management command
- `apps/payouts/services/reconcile_ledger.py::ReconcileLedgerService.execute() -> ReconcileReport`:
  ```python
  @dataclass
  class MerchantDrift:
      merchant_id: UUID
      issues: list[str]

  @dataclass
  class ReconcileReport:
      drifts: list[MerchantDrift]
      def is_clean(self) -> bool: return not self.drifts
  ```
  - For each merchant:
    - Query `held_paise` from `get_balance_breakdown()`.
    - `expected_held = SUM(payouts.amount_paise WHERE status IN (pending, processing) AND merchant_id=m)`.
    - If mismatch → record drift "held_mismatch: ledger=X, payouts=Y".
  - For each payout in terminal states, assert exactly the expected hold/release/debit txn count per spec §9 (1 hold + (1 debit XOR 1 release)).
- `apps/payouts/management/commands/reconcile.py`:
  - Calls service, prints report, returns exit code 0 or 1.
- **Test:** `tests/integration/test_reconciliation.py` — clean seeded data → exit 0; inject a `release` txn for a `completed` payout (drift) → exit 1 with drift listed.
- **Commit:** `feat(reconcile): ledger ↔ payout invariant checker + manage.py command`

### M5.5 — `stress_concurrency` harness
- `apps/payouts/management/commands/stress_concurrency.py`:
  - Args: `--merchants N`, `--workers W`, `--duration S`, `--per-merchant-credit P`.
  - Seed N merchants each with P credit. Spawn W threads each posting random-amount payouts to random merchants for `duration` seconds.
  - At end: run `ReconcileLedgerService`. Print summary (successes / 422s / 409s / errors). Exit non-zero if reconcile drifts.
- **Test:** `tests/integration/test_stress_smoke.py` — runs harness with tiny inputs (2 merchants, 4 workers, 2s) inside test DB, asserts clean.
- **Commit:** `feat(stress): concurrency harness for senior-signal demo`

## Verification Gate
- `pytest -v` → all green.
- `python manage.py reconcile` against seeded DB → exit 0.
- Manual: tail logs while creating a payout, capture into `logs.jsonl`, run `jq 'select(.payout_id == "...")' logs.jsonl` → reconstructs lifecycle.
