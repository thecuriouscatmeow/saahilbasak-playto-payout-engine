## Goal
Stand up Celery + Redis. Implement the worker that drives a payout from `pending` to terminal state, with bank-settlement simulation, atomic refund on failure, and sweeper-driven recovery for stuck payouts. Sweeper performs retries inline (spec §8 Option A).

## Acceptance Criteria
1. `process_payout` task moves a `pending` payout to `completed` (success path) with a `debit` txn in the same transaction.
2. Failure path moves to `failed` with a `release` txn — both committed atomically.
3. Hang path: payout left in `processing`; sweeper picks up after 30s and retries.
4. Max attempts (3) → atomic `failed` + `release`.
5. Duplicate task delivery is a no-op (worker first-line check + conditional UPDATE both prevent reprocessing).
6. Sweeper under concurrent invocations does not double-claim (`SKIP LOCKED` proven by test).

### M4.2 — `ProcessPayoutService`
- `apps/payouts/services/process_payout.py::ProcessPayoutService`:
  - `__init__(payout_id)`; `execute()`:
    1. Fetch payout. If `status != PENDING`: return `"already_handled"` (logged).
    2. `payout_repo.transition(frm=PENDING, to=PROCESSING, increment_attempt=True)`. If 0 rows: return `"raced"`.
    3. `outcome = simulate_bank_settlement()` — random: <0.70 success, <0.90 fail, else hang. Inject seed via `settings.BANK_SETTLEMENT_SEED` for tests.
    4. If hang: return `"hung"`.
    5. If success: `transition(frm=PROCESSING, to=COMPLETED, on_apply=lambda: txn_repo.insert_debit(payout, payout.amount_paise, reference="bank_settled"))`.
    6. If fail: `transition(frm=PROCESSING, to=FAILED, on_apply=lambda: txn_repo.insert_release(payout, payout.amount_paise, reference="bank_failed"))`.
- **Test:** `tests/integration/test_worker_success.py`, `test_worker_failure_refund.py`, `test_worker_hang.py` — each forces an outcome via patched `simulate_bank_settlement` and asserts final state, txn rows, balance.
- **Commit:** `feat(service): ProcessPayoutService with atomic refund on failure`

### M4.4 — `RetryStalePayoutsService` (sweeper)
- `apps/payouts/repositories/payout_repo.py::claim_stale_with_skip_locked(threshold_seconds=30, limit=100)`:
  ```python
  with connection.cursor() as cur:
      cur.execute("""
          SELECT id, attempt_count
          FROM payouts
          WHERE status = 'processing'
            AND last_attempted_at < now() - make_interval(secs => %s)
          ORDER BY last_attempted_at
          FOR UPDATE SKIP LOCKED
          LIMIT %s
      """, [threshold_seconds, limit])
      return cur.fetchall()
  ```
  Caller must hold an open transaction for the `SKIP LOCKED` to be meaningful.
- `apps/payouts/services/retry_stale.py::RetryStalePayoutsService.execute()`:
  ```python
  with transaction.atomic():
      rows = payout_repo.claim_stale_with_skip_locked(...)
      for payout_id, attempt_count in rows:
          if attempt_count >= 3:
              payout_repo.transition(payout_id, frm=PROCESSING, to=FAILED,
                  on_apply=lambda pid=payout_id: txn_repo.insert_release(...),
                  reason="max_attempts_exceeded")
          else:
              # Inline retry: bump last_attempted_at + attempt, run simulation, apply outcome
              Payout.objects.filter(id=payout_id).update(
                  last_attempted_at=Now(), attempt_count=F('attempt_count') + 1)
              outcome = simulate_bank_settlement()
              if outcome == 'success': transition(... COMPLETED, insert_debit ...)
              elif outcome == 'fail':  transition(... FAILED, insert_release ...)
              # 'hang' → leave for next sweep
  ```
- The whole loop runs in one atomic block so all claimed rows are released together.
- **Test:** `tests/integration/test_stale_sweeper.py`:
  - Insert a `processing` payout with `last_attempted_at = now() - 60s`, `attempt_count=2`. Force-fail outcome. Run service → assert `FAILED` + release.
  - `attempt_count=3` already → assert `FAILED` with `max_attempts_exceeded` reason in `PayoutEvent`.
  - Concurrency: 2 sweeper invocations in parallel (threads) → assert each row claimed exactly once (no double release).
- **Commit:** `feat(service): stale-payout sweeper with SKIP LOCKED + inline retry`
